# Changelog

## Unreleased

### Changed

- **Node labels are now two lines for leaf nodes.** The predicted class/value appears on the first line and the sample count `(n=...)` on the second. This cuts the horizontal label footprint roughly in half for leaf nodes, reducing overlap on wide trees.

- **Horizontal node spacing now adapts to label length.** Previously sibling nodes were always 72 px apart. Now the gap is computed from the longest label in the tree (`max_chars × 6.5 px + 24 px margin`), so the layout automatically widens when variable names or class names are long.

- **Nodes are now draggable.** Users can click and drag any node to manually reposition it. The connecting edges and edge-midpoint rule labels update in real time. Dragging a node does not affect the tree structure — it is purely a visual adjustment. Drag and zoom/pan can be used independently without interfering with each other.

### Developer notes

- Node label text is generated by `labelLines()` in `src/tree.ts`. Each element of the returned array becomes one line of text under the node. To display additional fields (e.g. impurity, depth, custom annotations), extend the returned array in that function. Leaf node lines go before the `(n=...)` entry; internal node lines go after the split condition. All fields on `TreeNode` (defined in `src/types.ts`) are available as `node.fieldName`.

```typescript
  // For leaf nodes, add extra lines before the (n=...) entry:
  return [node.predicted_class ?? "", `impurity=${node.impurity?.toFixed(3)}`, `(n=${node.n})`];

  // For internal nodes, add more lines after the split condition:
  return [splitLabel(node), `n=${node.n}`];
```
